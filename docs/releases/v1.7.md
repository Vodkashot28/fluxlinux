# Flux Linux v1.7 Release Notes

### 1. KDE GPU Renderer Picker
A new GPU renderer selection dialog has been added for KDE Plasma launches:
- **Turnip (Adreno Vulkan)**: Launches KDE using the Turnip Vulkan driver via Zink — best performance on Snapdragon devices with Adreno 6xx/7xx/8xx GPUs.
- **Software Rendering (LLVMpipe)**: Launches KDE with CPU-only software rendering — compatible with all devices, including those where GPU drivers cause black screens.
- **Removed VirGL**: The VirGL option has been dropped due to persistent black screen issues that could not be reliably resolved.
- **XDG_RUNTIME_DIR Fix**: Fixed a long-standing ownership conflict (`uid 1000` vs Termux uid) that caused dbus and KDE session failures.
- **Forced Compositor Disable**: `kwinrc` compositing is now force-disabled on every KDE launch to prevent rendering regressions.

### 2. AI / LLM Support — Vulkan llama.cpp + Qwen2.5-1.5B
Local AI inference is now available inside the Linux environment:
- **Vulkan llama.cpp**: Installs `llama.cpp` compiled with Vulkan GPU backend, enabling hardware-accelerated inference on Adreno GPUs via Turnip.
- **Qwen2.5-1.5B Model**: Downloads the `Qwen2.5-1.5B-Instruct` GGUF model (~938 MB) with integrity verification.
- **Auto-detection**: Launch script automatically detects available CPU/GPU and falls back gracefully if Vulkan is unavailable.
- **Home Screen Integration**: A "Run Qwen2.5-1.5B" button is available directly from the distro launch dialog.

### 3. Onboarding Improvements
- **In-App APK Downloads**: Step 1 of onboarding now downloads Termux and Termux:X11 APKs directly within the app, with a progress bar, and installs them via `FileProvider` — no need to navigate to a browser.
- **Auto-cleanup**: Downloaded APK files are automatically removed after installation.
- **Termux X11 Detection Fix**: Improved detection logic for Termux:X11 — resolves issue [#5](https://github.com/abhay-byte/fluxlinux/issues/5) where the app failed to detect X11 even after correct installation.
- **F-Droid Termux Notice**: Added a check and user-facing message to inform users that FluxLinux only supports the F-Droid version of Termux, not Play Store builds — addresses issue [#3](https://github.com/abhay-byte/fluxlinux/issues/3).

### 4. Themed Icon Support
- **Monochrome Adaptive Icon**: Added a monochrome layer to the adaptive icon, enabling proper themed icon support on Android 13+ (Material You / Dynamic Color).

### 5. Build & F-Droid Reproducibility
- **AGP Downgrade to 8.7.3**: Downgraded Android Gradle Plugin from 8.13.2 to 8.7.3 to fix F-Droid reproducible build failures caused by DEX binary mismatches between local and F-Droid build server toolchains.
- **Explicit F-Droid Build Entry**: Added a Builds entry for `versionCode 8` (v1.6) in the F-Droid metadata YAML — F-Droid requires an explicit entry per published version.
- **Corrected Signing Keys**: Fixed `AllowedAPKSigningKeys` in F-Droid metadata to match the actual `fluxlinux.jks` certificate fingerprint.
